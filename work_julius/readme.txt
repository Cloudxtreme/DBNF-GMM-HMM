work_julius


[はじめに]
Julius(dictation-kit-v4.3.1.-linux)を
動かすためのフォルダです。


[内容]
readme		:このファイルです。
*list		:認識にかけるファイルのパスです。
config		:Juliusのconfigファイルです。
model		:言語モデルと音響モデルです。
recog		:認識用のフォルダです。
sh		:実行スクリプトです。
segmentation	:transファイル作成用のフォルダです。


[使用方法]
認識結果を出力するまでの過程を説明します。

音響モデルと言語モデルを用意し、
modelフォルダ内のそれぞれのフォルダにおきます。

*listフォルダに、認識にかけるデータのパスを
記述します。

configファイルを設定します。

shフォルダ内のスクリプトを実行すると、認識結果が
recogフォルダ内の対応するフォルダに出力されます。

recog/scoringフォルダに認識結果を移動し、集計します。


[補足]
認識結果の集計方法ですが、
mkallhyp.shを実行することでlogファイルからhypファイルを作成します。
その際、本来なら拡張子が落とされるはずなのですが、
うまく落とされないものがありますので、手動で修正を
お願いします。
その後allscore.shを実行することで認識結果が集計されます。

なお、segmentationフォルダに関してですが、申し訳ありませんが
詳しい使用方法については省略させていただきます。
finishフォルダ内のtrain.transを作成することが目的です。
こちらのファイルをMatlabで処理することで、ラベルデータを
作成します。
